临时分支法解决git提交代码冲突

发表于 2019-10-20 分类于 Git操作系列 阅读次数：
git pull 获取代码时提示 rejected non-fast-forward 的错误解决

在改动git仓库中的文件时，没有在变更前执行 git pull 而导致本地的代码在后续的改动后和线上的代码不一致，而在commit后准备提交时再执行 git pull 获取代码时却提示 rejected -- non-fast-forward 的错误。

临时分支法

  1. 创建一个临时分支
  2. 将远端待提交分支拉取到本地的临时分支
  3. 手动将本地的待提交分支合并到临时分支上
  4. 将合并后的临时分支提交到远端的待提交分支
  5. 将远端分支拉取到本地，即将冲突合并后的代码更新到本地分支
  6. 删除临时分支
