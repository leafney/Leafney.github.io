我的家庭小主机配置-设备选型

发表于 2022-04-17 更新于 2022-04-18 分类于 家庭AIO小主机 阅读次数：
从0到1打造一款个人家庭All-in-One小主机，用技术让生活变得更美好

起因

其实一直都有想打造一款个人小主机的想法。

之前使用的是树莓派 raspberry-pi3 开发板，自己装了 docker 后跑了一些脚本和网站项目。

但是树莓派那仅仅1G的内存使用起来着实有些捉襟见肘，跑一些大型的服务就比较吃力。

后来在双十一的时候，花一百多购买了一年的阿里云1核2G的服务器。不过当到期再续费时年付竟然要八九百了，最后只得含泪备份数据，又换回了树莓派。

其实我的需求也很简单，就是想要一个能一直运行的linux电脑，然后跑一些脚本服务，或者搭建自己的Git仓库，部署个人的网站等等。在此基础上也可以进行一些扩展。

我家里有一台台式机，不过平时都是用笔记本，所以也就闲置了。但想想用台式机来实现我上面的需求，就感觉有些 大材小用 ，而且还很费电。测试过一段时间之后，光运行时的噪音就直接把我劝退了。

选择

后来在网上看了一些关于 低功耗小主机 的介绍，大家推荐比较多的就是 NUC 系列。

其中 NUC8 因为性价比高、还能通过加装网卡完美实现黑苹果，是大家推荐比较多的一款。

但我比较后发现和目前最新的 NUC11 相比下价格好像也差不了多少。不过比我的心理预期价位还是要高很多的。

想着就是跑一些服务之类的，也不用它来看视频、写代码，这样一想这个价格确实是不太超值。

最终，我把心理预期价位定在了 1500元 左右。

后来在B站上看到有介绍J4125软路由小主机的视频，顿时让我好像发现了新大陆一般。之前就曾听说过软路由，可以通过安装各种插件实现例如FQ、去广告的功能，觉得特别厉害。

记得之前将自己的路由器 网件R7000 特意刷了梅林固件想要实现FQ的效果，但最后发现限制想象的却是R7000那少得可怜的内存。

毕竟人就是这样嘛，有了之后，还想要更多。

总结了一下软路由相比起来有以下几点优势：

  • 网口多，接入家庭网络，可以实现更多功能
  • 支持内存和硬盘的自定义扩展
  • 软路由大部分是工控机，适合长时间24小时运行
  • 静音无风扇设计，不会影响晚上的睡眠

决定

综合以上的需求，我特意整理了以下的购买方案：

需求

  • 实现家庭软路由
  • 满足个人服务器需求，能运行docker容器或一些服务脚本
  • 需要长时间24小时开机运行，静音不影响晚上休息
  • 满足后期部署一些服务或个人项目，如homeassistant智能家居

购买方案

  • 方案一
    • 购买一个高配置微型PC，满足软路由+个人服务器需求
    • 配置要求至少内存16G，硬盘500G或以上
  • 方案二
    • 购买一个初级软路由设备，满足软路由需求
    • 再购买一个微型PC，实现个人服务器需求

其实电子产品发展还是很快的，有些前两年流行的配置现在来看已经有些勉强了。最后决定本着 买新不买旧 的想法，我选择了目前最新款的配置。

结尾

当设备开机的那一刻，恍然间我感觉，其实选择 16G内存和128GB的硬盘 就已经足够了。

不过话说回来，如果我换成 16G内存和128GB 的配置，价格在 1600元 左右，也算符合心理预期了。
